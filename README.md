# Sound Travels — Circumplex Rating Experiment

A jsPsych v7 experiment that plays short soundscape excerpts and asks participants to place each one on Russell's valence/arousal circumplex. Designed to run on **cognition.run** with audio + JS modules hosted on **GitHub Pages**.

## File layout

```
experiment/
├── index.html              minimal scaffold; loads everything else from BASE
├── manifest.json           generated by preprocess_audio.py — clip list, cluster IDs, features
├── audio/                  generated mp3 excerpts (12s, mono, 96 kbps)
├── css/
│   └── styles.css          all visual styling
└── js/
    ├── config.js           tunable knobs (trial counts, base URL)
    ├── circumplex-plugin.js custom jsPsych plugin: audio + 2D drag rating
    ├── trial-builder.js    stratified sampling + 3-phase ordering + practice clip pick
    ├── stages.js           every non-rating timeline node: consent, instructions,
    │                       preload, practice, breaks, demographics, debrief
    └── main.js             bootstrap: fetch manifest -> build timeline -> jsPsych.run
```

## Timeline (in order)

1. Consent
2. Instructions (with mini circumplex diagram)
3. Audio preload
4. **Practice trial** — one fixed clip, flagged `is_practice: true` and `phase: 0`. Excluded from analysis.
5. Phase 1 (8 trials, one per cluster, maximally diverse)
6. Break
7. Phase 2 (12 trials, mixed across clusters, anti-repeat constraint)
8. Break
9. Phase 3 (10 trials, all from one randomly-chosen cluster)
10. **Demographics page** — age, gender, native language, country, headphones, hearing impairment, years of musical training, hours/week of music listening, soundscape exposure
11. Debrief

## Pipeline overview

```
cluster_assignments.csv \

features.csv            /
                                                        |
```

## Step 1 — Preprocess audio (one-time, local)

From the parent `clusters_by_quality/` folder:

```bash
brew install ffmpeg          # if needed
python3 preprocess_audio.py
```

Generates `experiment/audio/*.mp3` and `experiment/manifest.json`. Roughly 30–50 MB total.

## Step 2 — Test it locally

```bash
cd experiment
python3 -m http.server 8000
```

Open `http://localhost:8000`. With `SOUND_TRAVELS_BASE = ""` (the default), all CSS, JS modules, audio, and manifest load from your local server. At the end you get a JSON download link instead of cognition.run capture.

## Step 3 — Push everything (audio + css + js + manifest) to GitHub

```bash
cd experiment
git init
git add audio css js manifest.json
git commit -m "experiment assets"
gh repo create sound-travels-circumplex --public --source=. --push
```

(Or create the repo manually on github.com and add a remote.) **Don't commit `index.html`** — that one piece lives on cognition.run.

In the GitHub UI: **Settings → Pages → Source: Deploy from a branch → main / (root)**. Wait ~60 s. Confirm with `https://USER.github.io/sound-travels-circumplex/manifest.json` in your browser.

## Step 4 — Wire up cognition.run

Open `index.html` and change exactly one line near the top:

```js
window.SOUND_TRAVELS_BASE = "https://USER.github.io/sound-travels-circumplex/";
```

(Trailing slash matters.) Then in cognition.run: **New experiment → Custom HTML / Advanced**, paste the entire `index.html` into the editor, save and publish. Open the participant URL yourself, run through it once.

That's it. You only ever paste this one HTML file into cognition.run — every change you make to the JS modules, CSS, or audio just needs `git push`, and the next participant gets the new version.

## Data shape

cognition.run automatically captures every `jsPsych.finishTrial({...})` call. In the downloaded CSV, one rating row looks like:

| field            | meaning                                                       |
|------------------|---------------------------------------------------------------|
| `clip_id`        | matches `manifest.json`                                       |
| `cluster`        | k-means cluster id (0–7)                                      |
| `phase`          | 0=practice, 1=warmup, 2=mixed, 3=within-cluster               |
| `phase_idx`      | trial index within the rated set                              |
| `is_practice`    | true on the practice trial only — filter these out            |
| `valence`        | placement on horizontal axis, normalized −1…+1                |
| `arousal`        | placement on vertical axis, normalized −1…+1                  |
| `n_plays`        | replays                                                       |
| `listened_full`  | true if they let it finish at least once                      |
| `rt_first_click` | ms from trial start to first marker placement                 |
| `rt_place_final` | ms from trial start to final placement                        |
| `rt_total`       | ms from trial start to clicking Next                          |
| `seed`           | per-session RNG seed (reproduces trial order)                 |
| `stage`          | string: `consent` / `instructions` / `practice` / `phase_1` / `phase_2` / `phase_3` / `break` / `demographics` / `debrief` |

The demographics row has its own shape: `response` is an object like `{age: "34", gender: "female", headphones: "yes", ...}` keyed by the form field names from `stages.js`.

To train the prediction model: filter to `phase` in (1, 2, 3), take the median (`valence`, `arousal`) per `clip_id` across participants as the supervised target, and join against the `features` block in `manifest.json` for X.

## Tweaking the experiment

Most things are one-line changes:

* trial counts → `js/config.js` (`N_PHASE_1`, `N_PHASE_2`, `N_PHASE_3`, `N_PRACTICE`)
* circumplex anchor labels (excited/tense/content/depressed) → `js/circumplex-plugin.js`
* demographic fields → `js/stages.js` `demographics()` function
* visual styling → `css/styles.css`
* cluster sampling strategy / clip count → `preprocess_audio.py` (`MAX_PER_CLUSTER`, `CLIP_SECONDS`)
* practice clip choice → `pickPracticeClip()` in `js/trial-builder.js`

After any of these changes (except the last, which requires re-running the Python script), `git push` and the next cognition.run participant gets the update.

## Troubleshooting

**Blank page in browser.** Open the JS console (Cmd+Option+J / F12). The most common causes: a 404 on `manifest.json` (run preprocess script), a 404 on a `js/*.js` file (your `SOUND_TRAVELS_BASE` is wrong), or jsdelivr blocked by an extension.

**"Failed to load manifest.json".** The error message in the page tells you the URL it tried. Visit that URL directly in your browser — if it 404s, your BASE is wrong.

**Audio doesn't autoplay on first trial.** Browser autoplay policy. The Play button always works as a manual fallback. The intro click usually unlocks autoplay for the rest of the session.

**Demographics form won't submit.** Required radio groups (headphones, hearing) must have a selection. The browser will highlight the missing field.
